## 1. `WorldOption.sav` 同步已停用

### 技術背景

Palworld 專服的設定寫入 `Pal/Saved/Config/WindowsServer/PalWorldSettings.ini`。  
但**已有存檔**的世界在啟動時，遊戲會同時讀取存檔目錄下的 `WorldOption.sav`；部分世界參數以 **SAV 為準**，會覆蓋或忽略 INI 中的對應值。

v1.2.0 中，GUI 儲存世界設定時**僅寫入 INI**；將 INI 轉為 `WorldOption.sav` 的 `palworld-worldoptions.exe` 流程已在 `setWorldSettingsiniByServerId.ts` **註解停用**（標註「退役了」）。

### 在什麼情況下會有問題

| 情境 | 是否受影響 | 說明 |
|------|------------|------|
| **全新伺服器**，建立後尚未有任何玩家進入過遊戲 | 通常不受影響 | 尚無 `WorldOption.sav`，INI 為唯一來源 |
| **已有存檔**（玩過一陣子、從 EA 升級到 1.0 沿用舊檔） | **高風險** | 玩家在 GUI 改設定後重啟，遊戲仍讀舊 SAV |
| 在 GUI 調整**倍率、語音、跨平台、死亡懲罰、PvP** 等世界規則 | **高風險**（有存檔時） | INI 已更新，但進遊戲後規則像「沒改」 |
| 僅修改**伺服器名稱、密碼、埠**（建立／編輯伺服器流程） | 影響較小 | 這些多在建立流程處理，且部分需重啟才生效 |
| 手動用記事本改 INI，且**刪除** `WorldOption.sav` 後重啟 | 不受本 bug 影響 | 屬手動 workaround，GUI 未提供此選項 |

### 開服時會遇到什麼

- 伺服器**可以正常啟動**，SteamCMD、埠、跨平台連線不受此問題阻擋。
- 房主在「世界設定」頁調整參數並按「確認變更」後，畫面上會顯示已儲存；**不會**出現錯誤提示。
- 若該世界**已有存檔**，重啟後實際生效的可能是**升級前或上次進遊戲時**鎖在 SAV 裡的舊規則，而非 GUI 剛寫入 INI 的新規則。
- 容易誤判為「GUI 壞了」或「1.0 更新把設定洗掉了」，其實是 **INI 與 SAV 不同步**。

### 遊玩時會遇到什麼

- **玩家體感**：房主宣稱已調高經驗倍率、開啟語音、改死亡懲罰等，進遊戲後**規則與說明不符**。
- **跨平台／語音**：在 GUI 開啟後，若 SAV 仍為舊值，可能出現「設定寫了但 Xbox／語音仍異常」的連鎖排查，根因其實是 SAV 未更新。
- **新開服且無人進過**：玩家通常**不會**察覺此問題；一旦有人進過並產生存檔，之後再改世界設定就可能踩雷。

### 暫時因應方式

1. **開新服、希望 GUI 設定完全生效**：在**第一次有玩家進入前**，於 GUI 調好世界設定並確認；避免先有存檔再大量改規則。
2. **已有存檔且要改規則**：關閉伺服器後，手動備份並視情況處理 `Pal/Saved/SaveGames/` 下對應世界的 `WorldOption.sav`（刪除後重啟會讓遊戲依 INI 重建，**可能影響該世界部分設定行為**，務必先備份整個 `SaveGames`）。
3. 或直接手動編輯 `PalWorldSettings.ini`，並了解 SAV 覆蓋風險（見 [Palworld 官方／社群 FAQ](https://gamesomg.com/palworld-server-settings-explained/) 中「設定不生效」說明）。

### 預計修復方向

- 恢復或改寫 `WorldOption.sav` 寫入流程（`convertToWorldOptionsSav` / Phase D）。
- GUI 儲存成功時，若偵測到已有存檔，**明確提示**「需同步 SAV 或刪除舊 SAV 後重啟才會生效」。

## 2. 啟動前無 Mod／引擎相容檢查

### 技術背景

Palworld 1.0 更新後，**遊戲引擎**、**UE4SS**、**Palguard** 以及使用者安裝的 `.pak` / Lua Mod 若仍為 Early Access 版本，是專服「按啟動後閃退」的常見原因。Pocketpair 亦建議 1.0 更新前**手動刪除**舊 Mod 檔（僅停用不足）。

v1.2.0 現況：

- **新建伺服器**預設 `ue4ssEnabled: true`、`palguardEnabled: true`（`createServerInstance.ts`）。
- **啟動按鈕**（`BootServerButton.tsx`）直接呼叫 `execStartServer`，**不做**引擎／UE4SS／Palguard 版本比對。
- 設定頁雖有 UE4SS／Palguard「一鍵更新」與 `EngineNeedInstall` 提示，但使用者**可以略過**且啟動時不會攔截。

### 在什麼情況下會有問題

| 情境 | 是否受影響 | 說明 |
|------|------------|------|
| **剛從 EA 一鍵更新到 1.0**，未更新 UE4SS／Palguard | **高風險** | 啟動後視窗閃退或秒退 |
| 新建伺服器後**直接按啟動**（預設開啟 UE4SS + Palguard） | **中～高風險** | 模板 DLL 與 1.0 引擎不匹配時失敗 |
| `Mods/`、`Paks/` 內留有 **EA 時期的 Mod** | **高風險** | 崩潰、連線異常、官方警告可能損壞存檔 |
| 已關閉 UE4SS、Palguard，且**無自訂 Mod** 的純原版服 | 通常不受影響 | 最常見的穩定路徑 |
| 僅使用 REST 管理、不開 Mod 框架 | 不受影響 | — |
| 在設定頁已將 UE4SS／Palguard **更新至與 GUI 模板一致** | 通常不受影響 | — |

### 開服時會遇到什麼

- 點「啟動伺服器」後**沒有明確錯誤對話框**；可能只看到程序視窗一閃即關。
- 房主需自行排查：關 Mod、更新 Palguard、重裝引擎、查 `Pal.log` 等，**除錯成本高**。
- `EngineNeedInstall` 僅在**全域引擎**過舊時彈窗，**不會**針對單一實例的 UE4SS／Palguard 在每次啟動前檢查。
- 1.0 更新後第一週開服高峰，此問題與 Pocketpair 官方「刪除舊 Mod」建議疊加，失敗率偏高。

### 遊玩時會遇到什麼

- 若勉強啟動成功但 Mod 不相容：遊戲中可能**間歇崩潰**、存檔時卡住、**部分玩家無法連線**。
- 玩家端 Mod 與伺服器不一致時，連線失敗；房主難以從 GUI 區分是 Mod 問題還是埠／跨平台問題。
- **純原版、無 Mod 的服**：遊玩過程通常穩定，此已知問題幾乎無感。

### 暫時因應方式

1. 升級到 1.0 後：**先關閉** UE4SS、Palguard，**移除** `Mods/` 與自訂 `Paks/`，確認能穩定開服後再逐一啟用。
2. 在「伺服器設定」頁檢查並執行 **引擎／UE4SS／Palguard 一鍵更新**（伺服器須關閉）。
3. 更新前備份 `Pal/Saved/SaveGames` 與 `PalWorldSettings.ini`。
4. 參考 README「Palworld 1.0 相關」：更新後先無 Mod 測試，再逐一啟用。

### 預計修復方向

- P3 **功能 2 Phase A**：`BootServerButton` 啟動前執行 `checkCompatibility`，列出過舊元件並提供「一鍵更新／仍要啟動」。
- 可選：新建伺服器預設改為關閉 Mod 框架，或首次啟動引導更新引擎。

## 3. 帕魯遊戲資料未更新至 1.0

### 技術背景

GUI 內「贈送帕魯」「帕魯圖鑑」等功能依賴 `assets/game-data/data/pals.json` 與 `assets/game-data/images/pals/`。

v1.2.0 現況：

- `pals.json` 約 **187** 筆；Palworld 1.0 官方約 **287** 隻帕魯（含 72 隻新增）。
- 圖示檔約 **169** 張，同樣未涵蓋 1.0 新增帕魯。

此問題**不影響**遊戲引擎、存檔或專服程序本身。

### 在什麼情況下會有問題

| 情境 | 是否受影響 | 說明 |
|------|------------|------|
| 一般開服、連線、倍率、REST 踢人廣播 | **不受影響** | 不依帕魯 JSON |
| 房主使用 GUI **「贈送帕魯給玩家」**（RCON／Palguard） | **受影響** | 列表缺少約 100 隻 1.0 新帕魯 |
| 瀏覽 GUI **帕魯圖鑑／選單** | **受影響** | 新帕魯無名稱／無圖示 |
| 遊戲內野生生成、繁殖、捕捉 1.0 新帕魯 | **不受影響** | 由遊戲伺服器處理，與 GUI 資料無關 |
| 玩家正常打怪、蓋家、解鎖科技 | **不受影響** | — |

### 開服時會遇到什麼

- **完全不妨礙** SteamCMD 安裝、一鍵更新、設定埠、跨平台、啟動 `PalServer.exe`。
- 房主若預期用 GUI 開服時就「送一隻 1.0 新帕魯」做活動，會發現**選單裡沒有該帕魯**。

### 遊玩時會遇到什麼

- 玩家在遊戲內**正常遇到、捕捉** 1.0 新帕魯，玩法不受影響。
- 若房主想透過管理工具**補償玩家**（贈送特定新 Boss／新區域帕魯）：GUI **無法選到**，需改用 Palguard RCON 手動輸入內部 ID，或暫時不用 GUI 贈帕功能。
- 圖鑑類 UI 可能顯示不完整，造成「工具資料與遊戲版本不一致」的印象，但不會導致伺服器崩潰。

### 暫時因應方式

1. 贈送 1.0 新帕魯：使用伺服器日誌／Palguard 文件查**內部 Pal ID**，透過廣播欄 RCON 指令手動發放（若 Palguard 支援）。
2. 遊戲內管理玩家以**踢人、封禁、廣播、倍率**等 REST 功能為主，避開 GUI 贈帕魯選單。
3. 追蹤後續版本更新 `assets/game-data`。

### 預計修復方向

- 更新 `pals.json`、圖示與相關 `items` 資料至 1.0 對照表（獨立資料 PR）。
- 可選：從 `palservergui.net` 或社群資料源定期同步版本號。

## 情境對照：誰會先踩雷？

| 你的使用方式 | 最可能遇到的問題 |
|--------------|------------------|
| 1.0 **全新開服**、無 Mod、只用預設與 REST | 通常順暢；日後**有存檔再改世界設定**時可能踩 #1 |
| **EA 舊檔**升 1.0，在 GUI 改倍率／語音 | **#1** 設定改了卻不生效 |
| 1.0 更新後**直接一鍵啟動**、仍開 UE4SS／Palguard | **#2** 閃退、起不來 |
| 留有 **EA 時期 .pak／Lua Mod** | **#2** 崩潰、連線異常 |
| 常用 GUI **贈帕魯**管理玩家 | **#3** 新帕魯送不出 |
| 玩家純連線遊玩、房主用 REST 管理 | 多半**無感**（除非 #2 導致服不穩） |
